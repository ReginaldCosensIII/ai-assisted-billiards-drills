# Documentation Update Checklist

Whenever merging a task or pull request, verify that the following documentation areas have been updated if affected by the changes:

- [ ] **Master Brief (`docs/master-brief/`)**: Does this change the core vision or success criteria?
- [ ] **Scope (`docs/scope/`)**: Does this move an item in/out of scope or alter phase boundaries?
- [ ] **Architecture (`docs/architecture/`)**: Does this change system design, data flows, or dependency topology?
- [ ] **ADRs (`docs/adrs/`)**: Has a new technical decision been made that requires an ADR?
- [ ] **Testing (`docs/testing/`)**: Does this introduce a new testing tool or change QA strategies?
- [ ] **Workflow (`docs/workflow/`)**: Does this alter how agents or developers contribute to the project?
- [ ] **Content (`docs/content/`)**: Were user-facing strings, standard formats, or instructional copy modified?
- [ ] **Root `README.md`**: Does the project status, environment setup, or top-level project description need adjustment?
