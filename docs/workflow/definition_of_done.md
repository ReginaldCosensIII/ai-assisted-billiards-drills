# Definition of Done (DoD)

For a task, PR, or session to be considered "Done" in this project, it must meet all applicable criteria below:

## General Quality
- [ ] The work satisfies all Acceptance Criteria defined in the original task.
- [ ] The code/content is clean, minimal, and professional (no unused placeholders or generic boilerplate).

## Documentation & Process
- [ ] The `docs/workflow/documentation_checklist.md` has been completed.
- [ ] Any required ADRs for technical decisions have been written and accepted.
- [ ] Commit messages are clean, imperative, and correctly scoped.

## Technical Standards (If applicable)
- [ ] Linter and formatter checks pass without warnings.
- [ ] Applicable tests have been written and all tests pass.
- [ ] Code has been reviewed by the designated Architect agent or Human lead.
