# docs/adrs/

This folder contains Architecture Decision Records (ADRs) — lightweight documents that capture significant technical decisions, their context, and reasoning.

## ADR Format

Each ADR should be a single Markdown file named `ADR-NNNN-short-title.md` and include:

- **Status**: Proposed | Accepted | Deprecated | Superseded
- **Context**: The situation that prompted the decision
- **Decision**: What was decided
- **Consequences**: Trade-offs and impact of the decision

## Current Status

**Empty — Foundation stage.** ADRs will be created as technology and architecture decisions are made during the scope and architecture phases.
