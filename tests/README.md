# tests/

This directory will contain integration tests, end-to-end tests, and any cross-cutting test utilities.

## Intended Scope

- Integration tests spanning `app/` and `backend/`
- End-to-end test suites for user-facing drill flows
- Shared test fixtures and helpers

> **Note:** Unit tests will live co-located with the source code they test inside `app/`, `backend/`, and `shared/`. This `tests/` directory is reserved for higher-level, cross-boundary tests.

## Current Status

**Empty — Foundation stage only.** Testing strategy will be defined in `docs/testing/` before any tests are written here.
